# ArcReel 改造候选清单

日期：2026-05-16

## 背景

当前阶段先试跑 ArcReel 原版功能。正式改造前，先记录一版候选改造范围；后续用户确认增删后，再单独排序并逐项实施。

## A. 产品主界面改造

1. 右侧任务控制台：显示正在处理、排队中、失败、已完成任务。
2. 任务日志面板：每个生成任务可展开查看执行日志、错误原因、耗时、调用模型。
3. 文件归档面板：统一查看小说源文件、剧本 JSON、角色图、场景图、分镜图、视频片段、音频、成片。
4. API / 模型快捷设置面板：快速查看当前供应商、模型、余额/用量、可用状态。
5. 项目工作台重构：把“小说 -> 章节 -> 剧集 -> 分镜 -> 视频 -> 成片”做成更清晰的生产线界面。

## B. 小说到动画工作流

6. 小说章节读取与切片：支持按章节、字数、剧情节点自动拆分输入。
7. 总管 AI 工作流编排：决定下一步生成什么、调用哪个子流程、如何拆任务。
8. 角色设定生成：从小说自动提取角色，生成角色卡、外貌、性格、服装、参考图提示词。
9. 场景设定生成：自动提取常用地点，生成场景卡、时代背景、光影、布景、参考图提示词。
10. 道具/线索设定生成：提取关键物品、法器、武器、信物、机关等。
11. 剧本改写：把小说章节改成适合短视频/动画的剧本文本。
12. 分镜拆解：按镜头、景别、运镜、动作、对白、时长拆成可生成的视频单元。
13. 首尾帧提示词生成：为每个镜头生成首帧、尾帧、参考图提示词。
14. 提示词润色器：把粗提示词转成适合不同模型的高质量提示词。

## C. 多 Agent 架构

15. Agent Runtime Provider 抽象：让系统不只绑定 Claude Agent SDK，后续可切 OpenAI Agents SDK。
16. Claude Runtime 保留适配：当前能力先不拆坏，作为第一个 runtime provider。
17. OpenAI Agents SDK Runtime：新增 OpenAI Agents SDK 适配层。
18. 子 Agent 角色定义：总管、角色设定 Agent、场景 Agent、分镜 Agent、提示词 Agent、视频 Agent 等。
19. 工具接口显式化：把 `.claude/skills` 里的脚本逐步改成后端可控工具 API。
20. Agent 执行历史与回放：能看到每个 Agent 做了什么、用了什么输入、产出了什么文件。

## D. 视频生成能力

21. 视频参数预设：镜头时长、比例、清晰度、模型、风格、运动强度、角色一致性参数。
22. 分镜批量生成视频：一键把多个分镜排队生成视频片段。
23. 视频失败重试与替换：单个镜头失败后可重试、换模型、换提示词、替换素材。
24. 视频版本管理增强：一个镜头保留多版视频，支持比较、回滚、选中最终版本。
25. 参考视频到分镜增强：继续强化当前 reference video 工作流，支持更稳定地拆镜头。

## E. 音频管线

26. 独立 TTS 人声：不依赖视频模型自带声音，为角色生成对白。
27. 角色声线管理：每个角色绑定声音、语速、情绪、音色。
28. 字幕生成与时间轴对齐：自动生成字幕，和对白/视频片段对齐。
29. 环境声 / 音效 / BGM：场景声、动作音效、背景音乐分轨管理。
30. FFmpeg 混音与最终合成：视频、对白、音效、BGM、字幕统一合成成片。

## F. 素材库增强

31. 全局素材库 UI 增强：更方便管理角色、场景、道具、风格图、参考视频。
32. 项目素材和全局素材同步：项目内资产可保存到全局库，全局库也可复用到项目。
33. 素材标签与搜索：按角色、场景、风格、模型、项目、时间搜索。
34. 角色一致性资料包：每个角色聚合文字设定、参考图、历史生成图、提示词模板。
35. 风格预设库：玄幻、都市、赛博、国风、水墨、写实、动漫等风格模板。

## G. 配置与成本

36. 模型能力矩阵：明确每个供应商支持什么：文生图、图生图、图生视频、视频时长、比例、价格。
37. 成本预估增强：在生成前预估本章、本集、全项目成本。
38. 实际花费统计：生成后按项目、模型、任务类型统计费用。
39. API Key 健康检查：自动测试 Key 是否可用、额度是否异常、模型是否可访问。
40. 多供应商降级策略：一个模型失败后按规则切换备用模型。

## H. 导出与成片

41. 剪映草稿导出增强：导出视频轨、字幕轨、音频轨、素材文件夹。
42. 一键成片导出：直接生成最终 mp4。
43. 分集管理：一个小说项目下管理多集，每集有独立状态和成片。
44. 发布包归档：每集输出成片、字幕、封面、标题、简介、提示词记录。

## I. 基础工程

45. WSL 本地开发稳定化：固定启动命令、环境变量、依赖安装、数据目录。
46. `.env` 配置检查页：启动时提示哪些配置缺失。
47. 数据迁移和备份：项目数据、数据库、素材库可备份恢复。
48. 测试与回归保护：每改一个核心流程，补对应测试，避免后续越改越乱。
49. 中文产品文案统一：把界面语言从“开发工具感”调整成“小说动画生产工作台”。
50. 新手引导项目：内置一个示例小说项目，用来验证完整流程。
