# ArcReel 改造候选清单

日期：2026-05-16

## 背景

ArcReel 原版流程已可跑通，但生成质量距离“AI 小说转动画工作流软件”还有明显差距。当前改造重点从 UI 转向内容生产质量：先解决小说理解、短剧改编、分镜表达、角色一致性，再考虑视频生成增强、音频管线和界面升级。

## 当前优先级判断

1. 第一优先级：小说到动画工作流质量改造。
2. 第二优先级：角色/场景/道具一致性与资产资料包。
3. 第三优先级：音频管线与时间轴同步。
4. 第四优先级：视频生成能力增强。
5. 暂缓：产品主界面大改、多 Runtime 架构、OpenAI Agents SDK 适配。

## A. 产品主界面改造（暂缓）

> 暂时不作为第一阶段目标。先保证内容质量链路成立，再做界面承载。

1. 右侧任务控制台：显示正在处理、排队中、失败、已完成任务。
2. 任务日志面板：每个生成任务可展开查看执行日志、错误原因、耗时、调用模型。
3. 文件归档面板：统一查看小说源文件、剧本 JSON、角色图、场景图、分镜图、视频片段、音频、成片。
4. API / 模型快捷设置面板：快速查看当前供应商、模型、余额/用量、可用状态。
5. 项目工作台重构：把“小说 -> 项目圣经 -> 分集剧本 -> 导演分镜 -> 视频 -> 成片”做成更清晰的生产线界面。

## B. 小说到动画工作流（第一优先级）

6. 小说章节读取与切片：从“硬按字数切片”升级为“章节识别 + 剧情节点识别 + 自然断点建议”。
   - 识别章节标题、场景转换、时间跳跃、冲突点、悬念点。
   - 切片前先给出候选断点，让用户确认。
   - 输出每集使用的原文范围与剩余文本，避免重复消费或漏读。

7. 总管 AI 工作流编排：保留当前 Claude Agent SDK 方案，但重做编排策略。
   - 总管不直接长篇阅读小说，而是负责状态检测、阶段选择、任务派发、用户确认和结果汇总。
   - 将流程改为：项目圣经 -> 分集规划 -> 单集短剧脚本 -> 导演分镜 JSON -> 资产生成 -> 分镜图 -> 视频。
   - 每阶段完成后都产生可审查文件，并要求用户确认“继续 / 重做 / 修改要求 / 跳过”。
   - 增加工作流状态机，明确每个项目当前卡在哪一阶段。

8. 角色设定生成：从“角色图”升级为“角色资料包”。
   - 输出角色身份、人物弧光、关系网、外貌锚点、服装锚点、标志物、禁止变化项。
   - 最小版先在 `project.json` 或新文件中保存结构化字段；后续再生成多张角色参考图。
   - 目标不是只生成一张漂亮图，而是保证后续每个镜头都能稳定复用同一个角色。

9. 场景设定生成：已有基础能力，但需要升级为“场景资料包”。
   - 当前已有 scene 资产和 scene_sheet 图。
   - 需要补充空间结构、时代/地域、光影基调、可复用布景元素、禁止出现元素。
   - 后续分镜 prompt 必须引用场景资料包，而不是只引用场景名。

10. 道具/线索设定生成：已有基础能力，但需要升级为“线索连续性资料”。
   - 当前已有 prop 资产和 prop_sheet 图。
   - 需要补充道具首次出现、剧情作用、视觉特征、前后状态变化。
   - 对悬疑、玄幻、复仇类小说，道具/线索要服务剧情伏笔，而不只是生成一张物品图。

11. 剧本改写：新增“短视频剧集脚本”中间层。
   - 不再从小说片段直接生成最终分镜 JSON。
   - 先把小说改编成适合短视频的 `episode_script.md`：开场钩子、场景、对白、动作、情绪、结尾悬念。
   - 再由 `episode_script.md` 转成系统可执行的 `episode_N.json`。

12. 分镜拆解：从字段填充升级为“导演分镜”。
   - 每个镜头增加叙事目的、观众注意力焦点、情绪曲线、上一镜连续性、下一镜转场意图。
   - 分镜 prompt 中加入景别、构图、镜头运动、光影、人物动作、微表情、画面层次。
   - 目标是让每个镜头都知道“为什么存在”，而不只是“画面里有什么”。

13. 首尾帧提示词生成：已有部分能力，但需要重新定位。
   - 当前 storyboard/grid/video prompt 已有首帧、宫格切块、分镜图等机制。
   - 后续应把首尾帧作为“镜头状态变化”来设计：起始构图、结束构图、人物位置变化、情绪变化。
   - 对支持首尾帧的视频模型，单独生成 first_frame_prompt / last_frame_prompt；不支持的模型降级为普通图生视频 prompt。

14. 提示词润色器：当前没有独立模块，需要新增“模型适配与质量润色层”。
   - 现状：prompt builder 会拼接字段和防崩短语，但没有独立的润色/评分/模型适配器。
   - 新增 PromptPolisher：输入导演分镜、项目圣经、角色资料包，输出适配目标模型的图片/视频 prompt。
   - 支持按模型差异输出：Veo / Seedance / Sora / Grok / Vidu 的不同提示词风格。
   - 增加 prompt 自检：是否缺角色锚点、是否缺动作、是否太抽象、是否含 BGM/字幕/水印等禁止项。

## B+ 新增：项目圣经与质量评审（新增第一优先级）

15. 项目圣经 `story_bible.json`。
   - 包含世界观、人物关系、角色弧光、视觉基调、镜头语言、禁忌项。
   - 后续剧本、分镜、资产图、视频 prompt 都必须读取它。

16. 分集策划 `episode_plan.md/json`。
   - 先规划每集剧情目标、开场钩子、高潮点、结尾悬念，再生成单集脚本。
   - 避免把小说机械切成一段段无短剧结构的素材。

17. 质量评审与返工机制。
   - 每阶段输出后自动给出检查清单。
   - 对剧本检查：是否有钩子、冲突、转折、悬念。
   - 对分镜检查：是否有动作、焦点、连续性、可生成性。
   - 对角色检查：是否有外貌锚点和禁止变化项。

18. 人工可编辑中间文件。
   - 项目圣经、分集计划、短剧脚本、导演分镜都应是可读可改的文件。
   - 后续重新生成时尊重用户修改，不覆盖人工判断。

## C. 多 Agent 架构（低优先级，暂缓 Runtime 改造）

19. Agent Runtime Provider 抽象：暂缓。先继续使用 Claude Agent SDK。
20. Claude Runtime 保留适配：保留现状，不先拆。
21. OpenAI Agents SDK Runtime：暂缓。
22. 子 Agent 角色定义：保留并加强当前项目已有的 subagent 分工。
   - 当前已有类似子 Agent：`analyze-assets`、`split-narration-segments`、`normalize-drama-script`、`split-reference-video-units`、`create-episode-script`、`generate-assets`。
   - 后续新增或重写：`build-story-bible`、`plan-episodes`、`write-episode-script`、`direct-storyboard`、`review-quality`。
23. 工具接口显式化：保留为中期目标。
   - 当前已有一部分生成能力走 MCP 工具。
   - 后续继续把确定性流程做成后端工具，而不是让 Agent 靠自由文本和脚本拼。
24. Agent 执行历史与回放：中期目标。
   - 需要能看到每个 Agent 做了什么、用了什么输入、产出了什么文件。

## D. 视频生成能力（第四优先级）

25. 视频参数预设：镜头时长、比例、清晰度、模型、风格、运动强度、角色一致性参数。
26. 分镜批量生成视频：一键把多个分镜排队生成视频片段。
27. 视频失败重试与替换：单个镜头失败后可重试、换模型、换提示词、替换素材。
28. 视频版本管理增强：一个镜头保留多版视频，支持比较、回滚、选中最终版本。
29. 参考视频到分镜增强：继续强化当前 reference video 工作流，支持更稳定地拆镜头。
30. 小样片验证模式：先生成 2-3 个关键镜头作为质量样片，通过后再批量生成整集。

## E. 音频管线（第三优先级）

31. 独立 TTS 人声：不依赖视频模型自带声音，为角色生成对白。
32. 角色声线管理：每个角色绑定声音、语速、情绪、音色。
33. 字幕生成与时间轴对齐：自动生成字幕，和对白/视频片段对齐。
34. 环境声 / 音效 / BGM：场景声、动作音效、背景音乐分轨管理。
35. FFmpeg 混音与最终合成：视频、对白、音效、BGM、字幕统一合成成片。
36. 主时间轴 `timeline.json`。
   - 明确每句对白的 speaker、text、start、end、emotion。
   - 音频模型不猜人物什么时候说话，而是按时间轴生成。
   - 一期先做旁白/字幕/BGM 同步；二期再做角色对白和 lip-sync。

## F. 素材库增强（第二优先级的一部分）

37. 角色资料包：每个角色聚合文字设定、参考图、历史生成图、提示词模板、禁止变化项。
38. 场景资料包：每个场景聚合空间结构、风格、光影、可复用布景元素、参考图。
39. 道具/线索资料包：每个道具记录视觉特征、剧情作用、状态变化。
40. 全局素材库 UI 增强：更方便管理角色、场景、道具、风格图、参考视频。
41. 项目素材和全局素材同步：项目内资产可保存到全局库，全局库也可复用到项目。
42. 素材标签与搜索：按角色、场景、风格、模型、项目、时间搜索。
43. 风格预设库：玄幻、都市、赛博、国风、水墨、写实、动漫等风格模板。

## G. 配置与成本（中低优先级）

44. 模型能力矩阵：明确每个供应商支持什么：文生图、图生图、图生视频、视频时长、比例、价格。
45. 成本预估增强：在生成前预估本章、本集、全项目成本。
46. 实际花费统计：生成后按项目、模型、任务类型统计费用。
47. API Key 健康检查：自动测试 Key 是否可用、额度是否异常、模型是否可访问。
48. 多供应商降级策略：一个模型失败后按规则切换备用模型。

## H. 导出与成片（后续）

49. 剪映草稿导出增强：导出视频轨、字幕轨、音频轨、素材文件夹。
50. 一键成片导出：直接生成最终 mp4。
51. 分集管理：一个小说项目下管理多集，每集有独立状态和成片。
52. 发布包归档：每集输出成片、字幕、封面、标题、简介、提示词记录。

## I. 基础工程（持续保障）

53. WSL 本地开发稳定化：固定启动命令、环境变量、依赖安装、数据目录。
54. `.env` 配置检查页：启动时提示哪些配置缺失。
55. 数据迁移和备份：项目数据、数据库、素材库可备份恢复。
56. 测试与回归保护：每改一个核心流程，补对应测试，避免后续越改越乱。
57. 中文产品文案统一：把界面语言从“开发工具感”调整成“小说动画生产工作台”。
58. 新手引导项目：内置一个示例小说项目，用来验证完整流程。

## 建议第一阶段实施包

1. 新增项目圣经 `story_bible.json`。
2. 改造小说分析，输出项目圣经 + 更强角色/场景/道具定义。
3. 新增分集策划与短视频剧集脚本 `episode_script.md`。
4. 改造第一集剧本生成，让 `episode_script.md` 转成 `episode_N.json`。
5. 扩展导演分镜字段：叙事目的、视觉焦点、情绪曲线、连续性、转场意图。
6. 改造分镜/视频 prompt builder，消费项目圣经、角色资料包和导演字段。
7. 增加最小版质量评审，先不做复杂 UI，只在 Agent 输出和文件中呈现检查结果。

## 待确认
